## What It Does

Automatically generate beautiful release notes from your commit history when PRs are merged. Parses conventional commits (`feat`, `fix`, `docs`, etc.) and formats them into a clean, human-readable changelog — no more manually writing release notes.

## Features

- ✅ **Conventional Commits parsing** — automatically categorizes `feat`, `fix`, `docs`, `refactor`, and more
- ✅ **Custom templates** — use your own Markdown template with `{{changes}}` placeholder
- ✅ **Output flexibility** — get release notes as an action output for further processing
- ✅ **Summary counts** — know how many features, fixes, and docs changed at a glance
- ✅ **Auto-triggered** — runs on every merged PR, keeping changelogs perpetually up-to-date

## Quick Start

```yaml
name: Release Notes

on:
  pull_request:
    types: [closed]

jobs:
  release-notes:
    if: github.event.pull_request.merged == true
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: yw13931835525-cyber/release-notes-drafter-action@main
        with:
          base-ref: 'main'
          head-ref: ${{ github.head_ref }}
```

## Use Cases

### Generate Changelog on Every Merge
Keep your changelog fresh by automatically updating it every time a PR is merged to main.

```yaml
- name: Draft Release Notes
  if: github.event.pull_request.merged == true
  uses: yw13931835525-cyber/release-notes-drafter-action@main
  with:
    base-ref: 'main'
    head-ref: ${{ github.head_ref }}
    include-unlabelled: 'true'
```

### Custom Template for Your Brand
Use a custom template to match your project's release note style.

```yaml
- uses: yw13931835525-cyber/release-notes-drafter-action@main
  with:
    base-ref: 'main'
    head-ref: ${{ github.head_ref }}
    template: |
      ## 🚀 Release Notes

      {{changes}}

      ---
      *Generated by Release Notes Drafter*
```

## Configuration

| Input | Required | Default | Description |
|-------|----------|---------|-------------|
| `token` | No | `${{ github.token }}` | GitHub token with repo access |
| `base-ref` | No | — | Base branch to compare from (e.g., `main`) |
| `head-ref` | No | (current branch) | Head branch to compare to |
| `template` | No | (default) | Custom Markdown template with `{{changes}}` |
| `include-unlabelled` | No | `true` | Include commits without conventional labels |

## Outputs

| Output | Description |
|--------|-------------|
| `release_notes` | Generated release notes in Markdown |
| `changed_features` | Number of features changed |
| `changed_fixes` | Number of bug fixes |
| `changed_docs` | Number of documentation changes |

## Conventional Commits Format

This action parses commits following the [Conventional Commits](https://www.conventionalcommits.org/) specification:

```
<type>[optional scope]: <description>

[optional body]

[optional footer(s)]
```

### Supported Types

| Type | Description |
|------|-------------|
| `feat` | New features |
| `fix` | Bug fixes |
| `docs` | Documentation changes |
| `style` | Code style changes |
| `refactor` | Code refactoring |
| `perf` | Performance improvements |
| `test` | Test additions/changes |
| `build` | Build system changes |
| `ci` | CI/CD changes |
| `chore` | Maintenance |
| `revert` | Reverted commits |
